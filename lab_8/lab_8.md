## 1. Методика проведения анализа

### 1.1. Установка необходимых инструментов
Для проведения анализа уязвимостей с использованием OVAL были установлены следующие пакеты:
```bash

sudo apt install openscap-scanner openscap-utils
```

### 1.2. Загрузка OVAL-определений для Ubuntu 24.04 (Noble)
Были загружены актуальные OVAL-определения уязвимостей от Canonical:
```bash

wget https://security-metadata.canonical.com/oval/com.ubuntu.noble.usn.oval.xml.bz2
bunzip2 com.ubuntu.noble.usn.oval.xml.bz2
ls -la com.ubuntu.noble.usn.oval.xml  # Проверка наличия файла
```

### 1.3. Запуск сканирования системы
Сканирование системы выполнено с генерацией HTML-отчёта:
```bash

sudo oscap oval eval --report report_noble.html com.ubuntu.noble.usn.oval.xml
```

## 3. Описание наиболее серьёзных уязвимостей каждого типа

### 3.1. **Уязвимости ядра Linux (Linux kernel vulnerabilities)**
- **USN:** 7889-1, 7879-1–4, 7880-1, 7887-1–2
- **Ключевые CVE:** CVE-2025-21729, CVE-2025-38227, CVE-2025-38616, CVE-2025-38678
- **Описание:** Множественные уязвимости в ядре Linux, включая ошибки в подсистемах:
    - Сетевой стек (CVE-2025-38678)
    - Файловые системы
    - Драйверы устройств
- **Уровень риска:** Высокий — могут приводить к отказу служб, повышению привилегий, утечке данных.
- **Рекомендация:** Обновить ядро через `sudo apt update && sudo apt upgrade linux-image-*`.

### 3.2. **Уязвимости WebKitGTK**
- **USN:** 7914-1, 7895-1
- **Ключевые CVE:** CVE-2025-43434, CVE-2025-43392, CVE-2025-43443
- **Описание:** Уязвимости в движке WebKitGTK, используемом в браузерах и приложениях GNOME. Могут приводить к:
    - Выполнению произвольного кода
    - Утечке данных
    - Межсайтовому скриптингу (XSS)
- **Уровень риска:** Критический — влияет на браузеры и веб-приложения.
- **Рекомендация:** Обновить пакеты `webkit2gtk`, `libwebkit2gtk`.

### 3.3. **Уязвимости в библиотеках обработки изображений (ImageMagick)**
- **USN:** 7876-1, 7812-1, 7756-1, 7728-1
- **Ключевые CVE:** CVE-2025-62171, CVE-2025-57803, CVE-2025-55212
- **Описание:** Уязвимости в ImageMagick, используемом для обработки изображений. Могут приводить к:
    - Переполнению буфера
    - Выполнению кода при обработке специально сформированных изображений
- **Уровень риска:** Высокий — используется многими серверными и клиентскими приложениями.
- **Рекомендация:** Обновить пакет `imagemagick`.

### 3.4. **Уязвимости в мультимедийных библиотеках (FFmpeg)**
- **USN:** 7830-1, 7823-1, 7738-1, 7538-1, 6803-1
- **Ключевые CVE:** CVE-2023-6603, CVE-2025-7700, CVE-2024-36613
- **Описание:** Уязвимости в FFmpeg, используемом для обработки аудио и видео. Могут приводить к:
    - Отказу в обслуживании (DoS)
    - Выполнению произвольного кода
- **Уровень риска:** Высокий — используется многими медиаприложениями и сервисами.
- **Рекомендация:** Обновить пакеты `ffmpeg`, `libavcodec*`.

### 3.5. **Уязвимости в системных утилитах и службах**
- **PostgreSQL (USN-7908-1):** CVE-2025-12817, CVE-2025-12818
- **CUPS (USN-7912-1):** CVE-2025-58436
- **OpenVPN (USN-7898-1):** CVE-2025-13086
- **MySQL (USN-7873-1):** CVE-2025-53054, CVE-2025-53069
- **Уровень риска:** Средний/Высокий — зависит от использования служб.
- **Рекомендация:** Обновить соответствующие пакеты через `apt`.

## 4. Выводы по работе

1. **Методология анализа:**  
   Анализ проведён с использованием стандартного инструментария OpenSCAP и актуальных OVAL-определений от Canonical. Методика соответствует лучшим практикам анализа уязвимостей в системах Ubuntu.

2. **Общее состояние безопасности системы:**  
   Система находится в относительно хорошем состоянии: 753 из 783 проверок прошли успешно. Однако **29 уязвимостей требуют немедленного внимания**.

3. **Критические компоненты, требующие обновления:**
    - **Ядро Linux** — множественные уязвимости, высокий риск.
    - **WebKitGTK** — критично для веб-безопасности.
    - **ImageMagick и FFmpeg** — используются повсеместно, требуют срочного обновления.

4. **Рекомендации по устранению:**
    - Выполнить команду:
      ```bash
      sudo apt update && sudo apt upgrade
      ```
    - Проверить наличие обновлений для ядра:
      ```bash
      sudo apt install linux-image-generic
      ```
    - Перезагрузить систему после обновления ядра.
    - Регулярно проводить сканирование уязвимостей с помощью OpenSCAP.

5. **Профилактические меры:**
    - Настроить автоматическое обновление безопасности (`unattended-upgrades`).
    - Использовать только проверенные репозитории пакетов.
    - Регулярно анализировать логи безопасности.

6. **Заключение:**  
   Проведённый анализ показал, что система **Ubuntu 24.04.2 LTS** в целом соответствует требованиям безопасности, но требует установки ряда обновлений для устранения известных уязвимостей. Регулярное сканирование и своевременное обновление пакетов — ключевые меры для поддержания безопасной эксплуатации системы. Использование инструмента OpenSCAP позволяет автоматизировать процесс проверки уязвимостей и получать структурированные отчёты для анализа.
